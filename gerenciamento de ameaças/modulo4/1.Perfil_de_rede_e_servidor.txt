1. Perfilamento de rede
    duração de sessao
        Este é o tempo entre o estabelecimento de um fluxo de dados e seu término
    Taxa de transferencia total
        Este é a quantidade de dados que passam de uma determinada fonte para um determinado destino em um determinado período de tempo
    Portas usadas
        Este é uma lista de processos TCP ou UDP que estão disponíveis para aceitar dados
    Espaço de endereço de ativo crítico
        Este são os endereços de IP ou a localização lógica de sistemas ou dados essenciais.


2. Criação de perfil do servidor
    Portas de escuta
        Estes são os daemons e portas TCP e UDP que normalmente podem ser abertos no servidor
    Usuários e contas logadas
        Estes são os parâmetros que definem o acesso e o comportamento do usuário
    Contas de serviço
        Estas são as definições do tipo de serviço que um aplicativo pode executar
    Ambiente de software
        Essas são as tarefas, processos e aplicativos que têm permissão para serem executadas no servidor.


3. Detecção de anomalias de rede
    O comportamento da rede é descrito por uma grande quantidade de dados diversos, como os recursos do fluxo de pacotes, os recursos dos próprios pacotes e a telemetria de várias fontes. Uma abordagem para a detecção de ataques de rede é a análise desses dados diversos e não estruturados usando técnicas de análise de Big Data. Isso é conhecido como análise de comportamento de rede (NBA).

    ex.: Agora, o algoritmo pode ser interpretado como: a cada 5 minutos, obter uma amostragem de 1/100 dos fluxos durante o segundo 30. Se o número de fluxos for maior que 500, gere um alarme. Se o número de fluxos for inferior a 500, não faça nada. Este é um exemplo simples de usar um perfil de tráfego para identificar o potencial de perda de dados.


4. Teste de vulnerabilidade de rede 
    Testes periódicos se seg são essenciais para manter uma rede segura, como:
        análise de risco
            - Esta é uma disciplina na qual os analistas avaliam o risco representado por uma vuln para uma organização específica
            - Uma análise de risco inclui avaliação da probabilidade de ataques, identifica tipos de prováveis agentes de ameaça e avalia o impacto de exploração bem-sucedido na organização

        Avaliação de vulnerabilidade
            - Este teste emprega software para verificar servidores voltados para internet e redes internas em busca de vários tipos de vulnerabilidades
            - Essas vulnerabilidades incluem infecções desconhecidas, fraqueza nos serviços de banco de dados voltados para a web, patches de software ausentes, portas de escuta desnecessárias
            - As ferramentas para avaliação de vulnerabilidades incluem a plataforma OpsVAS de código aberto, Microsoft Baseline Security Analyzer, Nessus, Qualys e FireEye Mandiant serviços
            - A avaliação de vulnerabilidade inclui, mas vai além, varredura de portas

        Test de penetração
            - Este tipo de teste usa ataques simulados autorizados para testar a força de seg de rede.
            - Pessoal interno com experiência em hackers ou chakers éticos profissionais identificam ativos que podem ser albos de agentos de ameaças
            - Uma séria de exploração simulada são usadas com frequência
            - O teste de penetração não apenas verifica se existem vuln, mas também explora essas vuln para determinar o impacto potencial de uma exploração bem-sucedidade
            - Um teste de penetração individual é muitas vezes conhecido como pentest
            - Metasploit é uma ferramenta usada em testes de penetração
            - A core impact oferece software e serviços de pentest

    ex.: 
        análise de risco
            Os indivíduos realizam uma análise abrangente de impactos de ataques aos principais ativos da empresa e funcionando
                Consultores internos ou externo, Frameworks de gestão de risco
        vulnerabilidade avaliação
            Gerencimento de patches, varreduras de host, varredura de portas, outras varreduras e serviços de vuln
                OpenVAS, Microsoft Baseline Analyzer, Nessus, Qualys, Nmap
        pentest
            Uso de técnicas e ferramentas de hacking para penetrar defesas de rede e identificar a profundidade do potencial pentest
                Metasploit, CORE impact, Hackers éticos
